x402 · HTTP 402 live handshake click to see the protocol in action
The x402 protocol: an API endpoint returns HTTP 402 Payment Required with a challenge. The caller signs an EIP-3009 permit, retries with X-Payment header, the facilitator executes the 3-tx flow, the endpoint returns 200 with receipt.
This demo uses the real deployed MockUSDC + EscrowPayment contracts on Fuji. 
Click ① to make an unpaid request → server returns 402 Payment Required.
Click ② to sign a permit + retry → server returns 200 OK with receipt.
ERC-8004 · trustless agent reputation standard IERC8004 + per-category scores
ERC-8004 specifies a minimal interface for on-chain agent identity + reputation: getIdentity, getScore, getReputation. Our adapter exposes these over the deployed AgentRegistry + ReputationContract.
Plus we extend it with getCategoryScore — specialists score higher in their own category.
